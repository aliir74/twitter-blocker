# Twitter Hate Blocker: Non-Technical User Experience

**Date:** 2026-04-04
**Status:** Draft
**Goal:** Make the extension usable by Persian Twitter users with zero technical setup.

## Problem

The current extension requires users to: create an OpenRouter account, generate an API key, paste it in, choose a model, and configure a confidence threshold. This is a non-starter for non-technical users. Additionally, the extension is only available via developer mode sideloading, not the Chrome Web Store.

## Target Audience

Persian-speaking Twitter users dealing with hate speech on political/social posts. Heavily mobile but with desktop Chrome access. Not developer-savvy. Already using Telegram as their primary messaging app.

## Solution Overview

1. A **Cloudflare Worker + D1 backend** that holds the OpenRouter API key and handles AI classification server-side
2. A **simplified extension** with no API key setup, Persian language support, and a zero-config first-run experience
3. **Chrome Web Store** distribution for one-click install

The user experience becomes: install extension from store, visit a tweet, click the shield button, done.

## 1. Backend: Cloudflare Worker + D1

### Endpoints

**`POST /register`**
- Called once on extension first install
- Request: `{ clientId: string }` (UUID generated by extension)
- Extracts country from Cloudflare's `cf.country` request property (free, automatic)
- Inserts into `users` table
- Response: `{ ok: true, dailyLimit: 200 }`

**`POST /analyze`**
- The core classification proxy
- Request: `{ text: string, mainTweetText: string | null, blockingMode: "hate" | "cultPraise", clientId: string }`
- Validates `clientId` exists in D1
- Checks daily usage against rate limit (200/day)
- Calls OpenRouter with server-side API key (Gemma 3 12B, `google/gemma-3-12b-it`)
- Increments usage counters in D1
- Response: `{ isMatch: boolean, confidence: number, reason: string }`
- Error responses: `429` (rate limited), `400` (invalid request), `502` (OpenRouter error)

### D1 Schema

```sql
CREATE TABLE users (
  client_id TEXT PRIMARY KEY,
  country TEXT,
  created_at TEXT NOT NULL,
  last_seen_at TEXT NOT NULL
);

CREATE TABLE daily_usage (
  client_id TEXT NOT NULL,
  date TEXT NOT NULL,           -- YYYY-MM-DD
  requests INTEGER DEFAULT 0,
  flagged INTEGER DEFAULT 0,
  PRIMARY KEY (client_id, date),
  FOREIGN KEY (client_id) REFERENCES users(client_id)
);
```

### Server-Side Configuration

- **Model:** `google/gemma-3-12b-it` (hardcoded, best Persian support at lowest cost)
- **Prompts:** Hate speech and cult praise prompts move from extension to backend. This allows prompt iteration without shipping extension updates.
- **OpenRouter API key:** Stored as a Cloudflare Worker secret (`wrangler secret put OPENROUTER_API_KEY`)
- **Confidence threshold:** 90% (hardcoded server-side, not user-configurable)

### Rate Limiting

- 200 classifications/day per `clientId`
- Block All mode: tracked client-side only (no backend call). The 200/day limit applies only to AI classifications. Block All has its own client-side cap of 500/day stored in `browser.storage.sync` to prevent runaway automation.
- Check query: `SELECT requests FROM daily_usage WHERE client_id = ? AND date = ?`
- When limit hit, return `429` with `{ error: "daily_limit", resetsAt: "YYYY-MM-DDTHH:MM:SSZ" }`

### Abuse Prevention

- **Origin check:** `/analyze` endpoint validates `Origin` header matches the Chrome extension ID. No open CORS.
- **UUID farming:** Secondary IP tracking. If an IP generates 10+ `clientId`s, flag for review.
- **Prompts server-side:** Cannot be extracted or abused as a general-purpose LLM proxy.

### Cost Projection

| Users | Scans/day | Tokens/month | OpenRouter cost | Cloudflare |
|-------|-----------|-------------|-----------------|------------|
| 50    | 2,500     | ~15M        | ~$0.60          | Free tier  |
| 500   | 25,000    | ~150M       | ~$6             | Free tier  |
| 5,000 | 250,000   | ~1.5B       | ~$60            | Free tier  |

Assumes 50 scans/user/day average, ~200 tokens per request. Cloudflare free tier covers up to 5M D1 reads/day and 100k writes/day.

## 2. Extension Simplification

### Settings Popup: What Changes

**Removed (handled server-side):**
- API key input
- Model selector
- Confidence threshold slider

**Kept (simplified):**
- Blocking mode: Hate Speech / Cult Praise / Block All (radio buttons)
- Action mode: Block / Report / Both (radio buttons)
- Max replies: dropdown with 25, 50, 100 (instead of number input)
- Auto-scroll: on/off toggle
- Dry-run: on/off toggle

**Added:**
- Language toggle: English / Persian (auto-detected from `navigator.language` on first install)
- Usage indicator in popup footer: "34/200 scans used today"

### Settings Type Changes

```typescript
// Remove from Settings
- apiKey: string
- model: string
- confidenceThreshold: number
- maxScrollAttemptsWithoutNewContent: number

// Add to Settings
+ locale: "en" | "fa"
+ clientId: string

// Keep but simplify
  maxReplies: number  // constrained to 25 | 50 | 100
```

### clientId Lifecycle

1. On first install, extension generates `crypto.randomUUID()`
2. Stored in `browser.storage.sync` (persists across devices if user is signed into Chrome)
3. Calls `POST /register` to register with backend
4. Every `/analyze` call includes `clientId`

### Message Flow Change

Current:
```
Content Script → Background → OpenRouter API
```

New:
```
Content Script → Background → Your Cloudflare Worker → OpenRouter API
```

The background worker changes from calling OpenRouter directly to calling your backend. The content script is unchanged.

### What Stays the Same

- Content script DOM logic (scan, scroll, extract replies)
- Block/report execution (simulating clicks on Twitter UI)
- FloatingButton and LiveFeedPanel components
- All Twitter DOM selectors
- Auto-scroll with MutationObserver

## 3. Persian Language Support

### i18n Implementation

A simple object with `en` and `fa` keys. No framework needed.

```typescript
const i18n = {
  en: {
    scanReplies: "Scan Replies",
    scanning: "Scanning...",
    // ...
  },
  fa: {
    scanReplies: "اسکن ریپلای‌ها",
    scanning: "...در حال اسکن",
    // ...
  },
};
```

Locale detected from `navigator.language` on first install. User can toggle in settings. Stored in `browser.storage.sync`.

### RTL Handling

- When Persian is active, injected containers get `dir="rtl"` and `data-lang="fa"`
- CSS handles RTL layout via `[data-lang="fa"]` selectors
- Extension UI containers are isolated from Twitter's DOM, so RTL won't conflict with Twitter's layout

### What Stays in English

- LLM prompts (Gemma 3 12B already handles Persian content well)
- `reason` field in classification responses (brief diagnostic, not prominent in UI)
- Console logs

## 4. First-Run Onboarding

### Flow

1. User installs from Chrome Web Store (one click)
2. Extension generates `clientId`, calls `/register`
3. User opens any Twitter page, sees the floating shield button
4. First click on the button shows a lightweight overlay:
   - Brief explanation (1-2 sentences in detected language)
   - "This will scan replies and block hateful accounts for you."
   - "Continue" button
5. User clicks Continue, scan starts
6. Subsequent clicks skip the overlay and scan immediately

### Implementation

A `hasSeenOnboarding` boolean in `browser.storage.sync`. Checked by the content script before starting a scan. The overlay is a simple React component rendered in the existing panel container.

## 5. Chrome Web Store Distribution

### Store Listing

- **Name:** "پاک‌کن توییتر - Twitter Hate Blocker" (Persian first for discoverability)
- **Description:** Bilingual (Persian + English)
- **Screenshots:** Scan in action on a Persian tweet, results panel, settings
- **Category:** Social & Communication

### Privacy Policy (Required)

"No personal data is collected. An anonymous identifier tracks daily usage limits. Reply text from tweets you scan is sent to our classification server for AI analysis and is not stored after processing. Country is inferred from your IP for analytics only."

### Permissions

Same as current Manifest V3:
- `activeTab` (access current Twitter tab)
- `storage` (persist settings)
- Host permission for `twitter.com` and `x.com`
- Host permission for your Cloudflare Worker URL

### Shareable Stats

Results panel includes a "Copy stats" button that generates:
```
۴۵ ریپلای اسکن شد، ۱۲ اکانت بلاک شد 🛡️
#پاک‌کن_توییتر
```
Users can paste this as a tweet for organic distribution.
